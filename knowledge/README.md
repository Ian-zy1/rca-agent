## 二、设计原则

### 2.1 极简原则

每天只学 **1 个核心概念**,不贪多。1.5 小时 = 30 分钟理解 + 60 分钟动手。

### 2.2 问答驱动

每个知识点以 **Q&A** 形式组织,模拟:
- 比赛理论题(评委可能问的)
- 自测(检验是否真懂)

**问答标准**: 每个问题必须能用 **2-3 句话回答**,不能背诵式长篇。

### 2.3 动手优先

每个知识点配 **可跑代码**(硅基流动 API + Python),代码量控制在 30 行内,1.5 小时能跑通。

### 2.4 RCA 场景锚定

所有知识点都映射到 RCA 智能体的具体环节,不学用不上的。

## 四、7 天内容详细大纲(评审重点)

### Day 1 · LLM API 调用 + 核心名词

**学习目标**: 理解 LLM 是"文本输入/文本输出"的函数,能调用硅基流动 API

**核心 Q&A(5 个)**:
1. 什么是 Token?为什么计费按 Token 算?(1 个汉字 ≈ 1.5 token)
2. Context Window 是什么?超了会怎样?(7B 模型通常 32K,影响能塞多少文档)
3. Temperature 0 和 1 有什么区别?根因分析该用哪个?(0,要确定性)
4. System Prompt / User / Assistant 三个角色分别干什么?
5. 调用 LLM API 和调普通 REST API 有什么区别?(延迟高、流式、非确定)

**动手**: 注册硅基流动,跑通第一段 chat.completions 调用

**RCA 映射**: 这是整个 RCA 智能体的底层调用

---

### Day 2 · Prompt 结构 + Few-shot

**学习目标**: 会写结构化 Prompt,会用示例(Few-shot)控制输出

**核心 Q&A(5 个)**:
1. Prompt Engineering 是什么?为什么重要?(垃圾进垃圾出)
2. Few-shot 和 Zero-shot 区别?什么时候用 Few-shot?(任务复杂/格式严格)
3. Chain-of-Thought(CoT)是什么?RCA 为什么需要它?(多步推理)
4. Prompt 越长越好吗?有什么代价?(token 成本 + 注意力分散)
5. 如何让 LLM 稳定输出特定格式?(System Prompt 约束 + 示例引导)

**动手**: 写一个告警分类 Prompt,用 Few-shot 让模型输出 JSON

**RCA 映射**: 根因推断节点、报告生成节点

---

### Day 3 · 结构化输出 + LangChain 基础

**学习目标**: 让 LLM 输出程序能解析的结构化数据(JSON / Pydantic 对象)

**核心 Q&A(5 个)**:
1. 为什么不能直接用字符串解析 LLM 输出?(不稳定,可能带额外文字)
2. Pydantic 是什么?为什么和 LLM 配合好?(类型校验 + Schema 生成)
3. LangChain 的 Chain 是什么?和直接调 API 区别?(组合 Prompt + LLM + 解析)
4. `response_format={"type":"json_object"}` 有什么用和限制?
5. LLM 输出不符合 Schema 怎么办?(重试 + 校验 + fallback)

**动手**: 用 LangChain + Pydantic 输出一个 RCAReport 对象

**RCA 映射**: 报告生成节点(必须输出结构化)

---

### Day 4 · Function Calling ⭐(最关键)

**学习目标**: 让 LLM 自主决定调用哪个函数、传什么参数

**核心 Q&A(5 个)**:
1. Function Calling 本质是什么?(LLM 返回"要调哪个函数+参数",而不是自己执行)
2. 工具定义的 JSON Schema 哪些字段必须?(name, description, parameters)
3. LLM 怎么知道什么时候该调用工具?(靠 description 描述 + 上下文匹配)
4. 一次 Function Calling 的完整流程是什么?(问 → 模型返回 tool_call → 你执行 → 结果回传模型 → 最终回答)
5. Function Calling 和 ReAct Agent 区别?(FC 是单步;ReAct 是"思考-行动"循环)

**动手**: 定义 query_prometheus 和 query_loki 两个工具,让模型自动调用

**RCA 映射**: 指标分析、日志分析节点(整个 RCA 的发动机)

---

### Day 5 · Embedding + 向量检索

**学习目标**: 理解"语义搜索",会用 Chroma 存取向量

**核心 Q&A(5 个)**:
1. Embedding 是什么?为什么能做语义搜索?(文本→高维向量,相似文本距离近)
2. Embedding 维度多少?和效果什么关系?(768/1024/1536,越高越精细越贵)
3. 余弦相似度和欧氏距离区别?文本检索用哪个?(余弦,关注方向不关注绝对值)
4. 向量库(SQL like vs Chroma)有什么本质区别?(语义匹配 vs 字面匹配)
5. 为什么"Pod 内存溢出"和"OOMKilled"用 SQL like 搜不到,但向量能搜到?

**动手**: 存 10 条历史故障,输入"内存不够被杀"看 Top-3 检索

**RCA 映射**: 历史案例检索节点

---

### Day 6 · LangGraph 状态机 ⭐⭐(RCA 的骨架)

**学习目标**: 理解状态机式编排,会用 LangGraph 画工作流图

**核心 Q&A(5 个)**:
1. LangGraph 和 LangChain 的 Chain 区别?(图 vs 链;支持循环/分支/并行)
2. State 是什么?为什么节点间要共享 State?(流转的数据载体)
3. Node 和 Edge 的关系?条件分支怎么写?(add_conditional_edges)
4. 为什么 RCA 适合用状态机?(多步骤、有依赖、可能回环重试)
5. LangGraph 的 checkpoint 是什么?为什么重要?(中断恢复,长流程不丢)

**动手**: 定义 3 个节点(receive → analyze → infer),跑通一个图

**RCA 映射**: 整个 RCA 工作流的骨架(8 个节点)

---

### Day 7 · RAG 完整链路 + 串联

**学习目标**: 把检索 + Prompt + LLM 串成完整知识库问答

**核心 Q&A(5 个)**:
1. RAG 是什么?解决了 LLM 的什么问题?(知识时效 + 幻觉 + 私有数据)
2. RAG 完整流程有几步?(加载→切分→向量化→存储→检索→拼接→生成)
3. Chunk 切分策略对效果影响多大?(太大检索不准,太小丢失上下文)
4. 为什么 RAG 会"检索到了但答错"?(拼接 Prompt 不当 / 检索 Top-K 太少)
5. RAG 和 Fine-tuning 什么时候用哪个?(RAG 改知识;FT 改行为)

**动手**: 把 Day5 检索 + Day6 图 + Day4 FC 串成最小 RCA

**RCA 映射**: 案例检索 + 根因推断的完整闭环
