# RCA 方法论 + AIOps（领域方法 + 2026 前沿）

> 两块：**RCA 方法论**讲"怎么系统化排查"（agent 的推理脊柱选型），**AIOps**讲"AI 怎么进入运维"（比赛理论 + 知识面）。含 **2025-2026 最新 RCA agent 研究**——证明你的设计跟得上前沿，不是 2023 老套路。

# 第一部分：RCA 方法论

## 四种方法，哪些适合自动化 agent

| 方法 | 怎么做 | 适合 agent 吗 |
|---|---|---|
| **时间线分析** | 把部署/告警/指标异常按时间排，看因果 | ✅ **最适配**——agent 的数据层天然产出时间线 |
| **假设-检验循环** | 列假设 → 查数据证伪/证实 → 收敛 | ✅ **ReAct 式 agent 的天然形态** |
| **5 Whys** | 连问 5 个"为什么"往下钻 | ✅ 适合"已有假设后"深挖；易确认偏误 |
| 鱼骨图/Ishikawa | 按 人/机/料/法/环/测 分类发散 | ⚠️ 当 prompt 模板生成假设可以，运行时结构用不上 |
| 故障树 FTA | 顶向下布尔逻辑树 | ⚠️ 适合**离线编码**已知故障模式做预过滤，不适合 agent 现场建 |

**选型**：你的 RCA agent 主脊柱用 **时间线分析 + 假设-检验循环**；5 Whys 作为"根因推断"节点内的深挖 prompt；鱼骨/FTA 是考试口述点，不进 demo。

> 这是 PLAN.md「FC 用于节点内、节点间由图决定」的方法论依据——**假设-检验循环 = 节点内 FC 自由探索；图固定边 = 可控流程**。

## 假设-检验循环（agent 推理脊柱）

```
症状进来（告警/提问）
   ↓
生成 N 个假设（基于场景→指标对照表 + 历史案例）
   ↓
循环：挑一个假设 → FC 查数据 → 证实/证伪
   ↓ 收敛
置信度够 → 出根因报告
```
对应 LangGraph：拓扑节点生成假设，指标/日志节点 FC 查证，根因节点收敛。

# 第二部分：AIOps（AI + IT 运维）

## Gartner 定义（必考）

> AIOps = **大数据 + ML/LLM 增强或替代 IT 运维核心职能**：可用性与性能监控、事件关联分析、故障识别、自动化处置。

**5 大能力**：①跨域数据接入 ②拓扑感知 ③事件关联去噪 ④故障识别 ⑤处置增强。

## AIOps vs 普通 LLM 应用的本质区别

| | 普通 LLM 应用 | AIOps |
|---|---|---|
| 输入 | 用户文本 | **时序指标 + 日志 + 事件 + 拓扑**（多源异构）|
| 核心能力 | 生成/对话 | **异常检测 + 因果推断 + 关联去噪** |
| 输出 | 文本 | 根因 + 处置 + 预测 |

**关键**：AIOps 不是"ChatGPT 看仪表盘"，而是**统计/拓扑 + LLM 的混合**。这也是为什么你的 agent 要"拓扑关联 + 指标分析"双节点——纯 LLM 会幻觉，要结构化数据约束。

## 异常检测（AIOps 核心算法）

| 方法 | 原理 | 何时用 |
|---|---|---|
| **静态阈值** | `cpu > 90%` | 简单但误报多 |
| **z-score** | `(x−mean)/std`，>3σ 报警 | 基础，便宜 |
| **modified z-score (MAD)** | 用中位数+MAD，抗异常值 | 比 z-score 稳 |
| **STL 分解** | 拆 趋势+季节+残差，对残差报警 | **有日/周周期的指标必备**（多数运维指标有）|
| **ML**（Isolation Forest/AutoEncoder）| 学习正常模式 | 大规模、复杂，成本高 |

> 你的 RCA agent demo 可加个**轻量异常检测前置节点**：`rate(cpu[5m])` 的 z-score 超阈值才触发详细 RCA——这就是 AIOps 的"故障识别"能力落地。

## 告警关联去噪（AIOps 卖点 #1）

一次故障可能触发 50 条告警。**按 时间邻近 + 共享拓扑（同主机/同服务）聚成 1 个 incident**。这就是 PRD"告警聚合"节点的理论依据。

## 2026 商业 AIOps 格局（知识面）

| 厂商 | 引擎 | 特点 |
|---|---|---|
| **Dynatrace Davis** | 确定性因果 AI（Smartscape 拓扑）| **RCA 最强**；2026.6 新增 Claude 驱动的问题分析技能 |
| **Datadog Watchdog** | ML 相关分析 | 广而浅 |
| **New Relic iRCA** | 拓扑图+因果模型+路径排序 | 2026.2 预览版 |
| **BigPanda/Moogsoft** | 独立关联引擎 | 专一做去噪 |

趋势：**确定性拓扑 + LLM 推理** 混合成主流（Davis+Claude、iRCA 都是这路）。

# 第三部分：2025-2026 RCA Agent 前沿研究（证明你的设计跟得上）

> 这些论文证明"纯 ReAct 不够、拓扑+LLM 混合 + 树搜索"是前沿共识。比赛展示时能说"参考了 2026 最新研究"。

| 论文 | 方法 | 关键结论 |
|---|---|---|
| **Flow-of-Action** (2025.2) | SOP 增强多智能体 | 准确率 **64% vs ReAct 35.5%** |
| **EoG** (2026) | 依赖图上的溯因推理（确定性控制器+LLM 局部取证）| 比 ReAct **F1 高 7×** |
| **MetaRCA** (2026) | 离线建元因果图，在线实例化 | 生产数据 **+29pp 准确率** |
| **LATS-RCA** (2026) | 语言 agent 树搜索根因假设 | LO2 基准 **91.3%**，生产 65.1% |
| **OpsAgent** (2025.10) | 自演化多智能体，**只用 14B 模型**，联想生产部署 | 小模型也能打 |

**四条经验共识**（前沿给你背书）：
1. **纯 ReAct 不够**（多篇报告 ReAct 仅 30-40%）
2. **拓扑/确定性 + LLM 混合是赢家**（EoG/MetaRCA/Davis 都是）
3. **树搜索 > 线性推理**（RCA 分支多，一条线查不全）
4. **小模型能上生产**（OpsAgent 14B）

> 你选 **LangGraph 固定图（确定性骨架）+ 节点内 FC（LLM 灵活取证）+ 场景对照表（结构约束）**，正好踩在"拓扑/确定性 + LLM 混合"这条前沿共识上。**这不是落后，是主流。**
